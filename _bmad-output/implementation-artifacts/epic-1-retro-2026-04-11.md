## What Didn't Go Well

### FastMCP Schema Coercion Drift (recurring)
FastMCP silently coerces tool parameter types before handlers run. This caused silent validation bypass when setter tools declared explicit types (`int`, `bool`) — `"5"` became `5`, `True` became `1`, masking service-layer validation failures.

- First appeared in **Story 1.3** (audio EQ setters). Required 3 passes to resolve.
- Reappeared in **Story 1.4** (`set_play_mode`, `set_sleep_timer`). Should not have reached review.

**Resolution pattern discovered:** Use `Annotated[object, Field(json_schema_extra={"type": "integer"})]` to preserve raw inputs for service-layer validation while keeping the explicit MCP-visible schema type.

### Multi-Pass Review Cycles
3 of 4 stories required post-review follow-up commits:
- Story 1.1: 4 patches (spurious `play_mode` write, silent unknown-mode fallback, bare `str` typing, missing all-None test)
- Story 1.3: schema coercion drift fix (3 passes)
- Story 1.4: coercion drift repeat for play mode and sleep timer setters

Root cause: no team-level convention for FastMCP tool parameter handling; no story template checklist item for schema-type contract tests.

---

## Lessons Learned

1. **FastMCP coercion is a framework risk, not a one-off.** Every control tool that requires service-layer validation is vulnerable. The `Annotated[object, ...]` fix pattern is the team convention going forward.

2. **Contract tests are the enforcement mechanism for schema drift** — but only if they assert parameter types upfront. Adding them retroactively (Story 1.4) works, but the guard should be a first-class story deliverable.

3. **Post-write authoritative state pattern** (established Story 1.2): after a write operation, re-read the device state rather than synthesizing a response from the request. Prevents stale or rounded responses. Held for sleep timer and EQ setters.

4. **Hardening stories work** — Story 1.4 generalized the Story 1.3 fix pattern into regression tests and transport parity coverage that will protect the entire epic's surface going forward.
